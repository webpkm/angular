<h2 id="Object-oriented_programming">Object-oriented programming</h2>

<p>Object-oriented programming (OOP) is a programming paradigm that uses <a class="glossaryLink" href="/en-US/docs/Glossary/abstraction" title="abstraction: Abstraction in computer programming is a way to&nbsp;reduce complexity and allow efficient design and implementation in complex software systems. It hides the technical complexity of systems behind simpler&nbsp;APIs.">abstraction</a>&nbsp;to create models based on the real world. OOP uses several techniques from previously established paradigms, including <a class="new glossaryLink" href="/en-US/docs/Glossary/modularity" title="The definition of that term (modularity) has not been written yet; please consider contributing it!">modularity</a>, <a class="glossaryLink" href="/en-US/docs/Glossary/polymorphism" title="polymorphism: Polymorphism is the presentation of one interface for multiple data types.
 For example, integers, floats, and doubles are implicitly polymorphic: regardless of their different types, they can all be added, subtracted, multiplied, and so on.">polymorphism</a>, and <a class="glossaryLink" href="/en-US/docs/Glossary/encapsulation" title="encapsulation: Encapsulation is the packing of data and functions into one&nbsp;component (for example, a class) and then controlling access to that&nbsp;component to make a &quot;blackbox&quot; out of the object.&nbsp;Because of this, a user of that class only needs to know its interface (that is, the data and functions exposed&nbsp;outside&nbsp;the class), not the hidden implementation.">encapsulation</a>. Today, many popular programming languages (such as Java, JavaScript, C#, C++, Python, PHP, Ruby and Objective-C) support OOP.</p>

<p>OOP&nbsp;envisions software as&nbsp;a collection of cooperating objects rather than&nbsp;a collection&nbsp;of functions or simply a&nbsp;list&nbsp;of commands (as is the traditional view). In OOP, each object can receive messages, process&nbsp;data, and send&nbsp;messages to other objects. Each object can be viewed as an independent little machine with a distinct role or responsibility.</p>

<p>OOP promotes greater flexibility and maintainability in programming, and is widely popular in large-scale software engineering. Because OOP strongly emphasizes&nbsp;modularity, object-oriented code is simpler to develop and easier to understand later on. Object-oriented code promotes&nbsp;more direct analysis, coding, and understanding of complex situations and procedures than less modular programming methods.<a href="#cite-1"><sup>1</sup></a></p>

<h2 id="Prototype-based_programming">Prototype-based programming</h2>

<p>Prototype-based programming is an OOP model&nbsp;that doesn't use&nbsp;classes, but rather accomplishes&nbsp;behavior reuse (equivalent to&nbsp;inheritance in class-based languages) by decorating (or expanding upon) existing <em>prototype&nbsp;</em>objects. (Also called&nbsp;classless, prototype-oriented, or instance-based programming.)</p>

<p>The original (and most canonical) example of a prototype-based language is <a class="external external-icon" href="http://en.wikipedia.org/wiki/Self (programming language)" title="Self">Self</a> developed by David Ungar and Randall Smith. However, the class-less programming style grows&nbsp;increasingly popular lately, and has been adopted for programming languages such as JavaScript, Cecil, NewtonScript, Io, MOO, REBOL, Kevo, Squeak (when using the Viewer framework to manipulate Morphic components), and several others.<a href="#cite-1"><sup>1</sup></a></p>

<h2 id="JavaScript_object_oriented_programming">JavaScript object oriented programming</h2>

<h3 id="Namespace">Namespace</h3>

<p>A namespace is a container which allows developers to bundle up&nbsp;functionality under a unique, application-specific name. <strong>In JavaScript a namespace is just another object containing methods, properties, and objects.</strong></p>

<div class="note">
<p><strong>Note:</strong> It's important to note that in JavaScript, there's no language-level difference between regular objects and namespaces. This differs from many other object-oriented languages, and can be a point of confusion for new JavaScript programmers.</p>
</div>

Use of namespaces also reduces&nbsp;the chance of name conflicts in an application, since each application's objects are properties of an application-defined global object.</p>

<h3 id="Custom_objects">Custom objects</h3>

<h4 id="The_class">The class</h4>

<p>JavaScript is a prototype-based language and&nbsp;contains no <code>class</code> statement, such as is found in C++ or Java. This is sometimes confusing for programmers accustomed to languages with a <code>class</code> statement. Instead, JavaScript uses functions as classes. Defining a class is as easy as defining a function. In the example below we define a new class called Person.</p>

<pre class="brush: js  language-js" data-number=""><code class=" language-js"><span class="token keyword">var</span> Person <span class="token operator">=</span> <span class="token keyword">function</span> <span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span><span class="token punctuation">}</span><span class="token punctuation">;</span><div class="line-number" data-start="1" style="top: 0px;"></div></code></pre>

<p>To create a new instance of an object <code>obj</code> we use the statement <code>new obj</code>, assigning the result (which is of type <code>obj</code>) to a variable to access it later.</p>

<p>In the example above we define a class named <code>Person</code>. In the example below we create two instances (<code>person1</code> and <code>person2</code>).</p>

<h4 id="The_methods">The methods</h4>

<p>Methods are functions (and defined like functions), but otherwise&nbsp;follow the same logic as properties. Calling a method is similar to accessing a property, but you add <code>()</code> at the end of the method name, possibly with arguments. To define a method, assign a function to a named property of the class's <code>prototype</code> property. <p>As that example shows, all of the references we have to the <code>sayHello</code> function— the one on <code>person1</code>, on <code>Person.prototype</code>, in the <code>helloFunction</code> variable, etc.— refer to the <em>same function</em>. The value of <code>this</code> during a call to the function depends on how we call it. Most commonly,&nbsp;when we call <code>this</code>&nbsp;in an expression where we got the function from an object property— <code>person1.sayHello()</code>— <code>this</code> is set to the object we got the function from (<code>person1</code>), which is why <code>person1.sayHello()</code> uses the name "Alice" and <code>person2.sayHello()</code> uses the name "Bob". But if we call it other ways, <code>this</code> is set differently: calling <code>this</code>&nbsp;from a variable— <code>helloFunction()</code>— sets <code>this</code> to the global object (<code>window</code>, on browsers). Since that object (probably) doesn't have a <code>firstName</code> property, we end up with "Hello, I'm undefined". (That's in loose mode code; it would be different [an error] in <a href="/en-US/docs/Web/JavaScript/Reference/Strict_mode" title="/en/docs/Web/JavaScript/Reference/Functions_and_function_scope/Strict_mode">strict mode</a>, but to avoid confusion we won't go into detail here.) Or we can set <code>this</code> explicitly using <code>Function#call</code> (or <code>Function#apply</code>), as shown at the end of the example.</p>

<h4 id="Inheritance">Inheritance</h4>

<p>Inheritance is a way to create a class as a specialized version of one or more classes (<em>JavaScript only supports single inheritance</em>). The specialized class is commonly called the <em>child</em>, and the other class is commonly called the <em>parent</em>. In JavaScript you do this by assigning an instance of the parent class to the child class, and then specializing it. In modern browsers you can also use <a href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/create#Classical_inheritance_with_Object.create" title="/en-US/docs/JavaScript/Reference/Global_Objects/Object/create#Classical_inheritance_with_Object.create">Object.create</a> to implement inheritance.</p>

<p><strong>Note:</strong> JavaScript does not detect the child class <code>prototype.constructor</code> (see <a href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/prototype">Object.prototype</a>), so we must state that manually. See the question "<a class="external external-icon" href="http://stackoverflow.com/questions/8453887/why-is-it-necessary-to-set-the-prototype-constructor">Why is it necessary to set the prototype constructor?</a>" on Stackoverflow.</p>

<h4 id="Encapsulation">Encapsulation</h4>

<p>In the previous example, <code>Student</code> does not need to know how the <code>Person</code> class's <code>walk()</code> method is implemented, but still can use that method; the <code>Student</code> class doesn't need to explicitly define that method unless we want to change it. This is called <strong>encapsulation</strong>, by which every class packages data and methods into a single unit.</p>

<p>Information hiding is a common feature in other languages often as private and protected methods/properties. Even though you could simulate something like this on JavaScript, this is not a requirement to do Object Oriented programming.<a href="#cite-2"><sup>2</sup></a></p>

<h4 id="Abstraction">Abstraction</h4>

<p>Abstraction is a mechanism that allows you to model&nbsp;the current part of the working problem, either by&nbsp;inheritance (specialization)&nbsp;or composition. JavaScript achieves specialization by inheritance, and composition by letting class instances be the values of other objects'&nbsp;attributes.</p>

<p>The JavaScript Function class inherits from the Object class (this demonstrates specialization of the model) and the Function.prototype property is an instance of Object (this demonstrates composition).</p>

<pre class="brush: js  language-js" data-number=""><code class=" language-js"><span class="token keyword">var</span> foo <span class="token operator">=</span> <span class="token keyword">function</span> <span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span><span class="token punctuation">}</span><span class="token punctuation">;</span>

<h4 id="Polymorphism">Polymorphism</h4>

<p>Just as&nbsp;all methods and properties are defined inside the prototype property, different classes can define methods with the same name; methods are scoped to the class in which they're defined, unless&nbsp;the two classes hold a parent-child relation (i.e. one&nbsp;inherits from the other in a chain of inheritance).</p>

<h2 id="Notes">Notes</h2>

<p>These are not the only ways you can implement object-oriented programming in JavaScript, which is very flexible in this regard. Likewise,&nbsp;the techniques shown here do not use any language hacks, nor do they mimic other languages' object theory implementations.</p>

<p>There are other techniques&nbsp;that make even more advanced object-oriented programming in JavaScript, but those are beyond the scope of this introductory article.</p>
